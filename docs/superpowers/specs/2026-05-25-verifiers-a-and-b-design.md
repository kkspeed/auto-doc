## 0. Why this sub-project exists

The parent design's success criteria SC4 and SC5 are about catching two failure modes mechanically before they propagate through the round loop:

- **SC4 (cite enforcement):** every `[^ev-NNNNNN]` in the doc resolves to a non-superseded `evidence/ev-NNNNNN.md`, and every assertion in a `decided` section has at least one such cite.
- **SC5 (excerpt match):** every cite's surrounding sentence approximately matches the cited evidence file's `excerpt` field, normalized for trivial formatting differences.

Sub-project 1's pre-commit hook already catches some of these at git-commit time. This sub-project lifts the same logic into pure-Python library functions that the orchestrator (sub-project 4) can call at the verifier phase of the round state machine — before any commit happens. The two layers are intentional defense-in-depth: verifiers catch failures inside the round and produce a structured failure record the orchestrator writes as `rj-*.md`; the hook is the final commit-time gate. Verifier A and Verifier B never block a git commit themselves — that's the hook's job.

## 2. Public API surface

```python
from dataclasses import dataclass, field
from pathlib import Path


@dataclass
class VerifierFailure:
    kind: str           # "uncited-claim" | "dangling-cite" | "superseded-cite" | "excerpt-mismatch"
    variant: str        # "v-001"
    section_path: str   # "variants/nodes/v-001/doc/01-retry-policy.md"
    detail: str         # human-readable explanation
    excerpt_diff: str | None = None   # populated for excerpt-mismatch only


@dataclass
class VerifierResult:
    verdict: str        # "pass" | "fail"
    failures: list[VerifierFailure] = field(default_factory=list)


def verify_citation_completeness(
    variants_nodes_root: Path,
) -> VerifierResult:
    """For every `decided`-tagged section in every variant, verify every
    assertion sentence has at least one [^ev-NNNNNN] cite."""


def verify_cite_resolution(
    variants_nodes_root: Path,
    evidence_root: Path,
) -> VerifierResult:
    """For every [^ev-NNNNNN] cite in every variant's doc sections, verify
    evidence/ev-NNNNNN.md exists AND its TOML frontmatter has no
    `superseded_by` field (or has an empty one)."""


def verify_excerpt_match(
    variants_nodes_root: Path,
    evidence_root: Path,
    threshold: float = 0.92,
) -> VerifierResult:
    """For every [^ev-NNNNNN] cite, extract the sentence containing the cite,
    load ev-NNNNNN.md's `excerpt` field, normalize both, compute
    difflib.SequenceMatcher ratio, pass if >= threshold."""
```

Internal helper (also unit-tested independently):

```python
def _normalize_text(s: str) -> str:
    """NFC + smart-quote/dash map + lowercase + whitespace collapse +
    per-word leading/trailing punctuation strip."""
```

## 3. Behavioral semantics

### 3.1 Walk semantics (shared by all three public verifiers)

Each verifier walks `variants_nodes_root.iterdir()`. For each variant directory whose name starts with `v-`:

1. Look at `<variant>/doc/*.md` (skip if the doc/ dir doesn't exist).
2. For each `.md` file, read text. Skip if the file doesn't start with `+++`.
3. Find the second `+++`; the text between is the frontmatter, the text after is the body.
4. Parse the frontmatter via `tomllib.loads(frontmatter_text)`. Skip on parse error (orchestrator validates frontmatter at write time; verifiers tolerate malformed files at runtime by skipping rather than crashing).
5. Read `tags` from the parsed frontmatter (default `[]`).
6. Body = the substring after the closing `+++` plus the newline.

The walk is read-only. Verifiers never modify files.

### 3.2 `verify_citation_completeness`

Operates only on sections with `"decided"` in `tags`.

**Body preprocessing:**

- Strip fenced code blocks: text between matching ` ``` ` markers (multiline) is removed before sentence-splitting.
- Strip ATX headings: any line starting with `#` (followed by space or end-of-line) is removed.
- The result is the "prose body."

**Sentence splitting:**

- Split the prose body by regex `r'(?<=[.!?])(?:\s+|\Z)'` — sentence-end punctuation followed by whitespace or end-of-string.
- A sentence is an **assertion** if (a) it ends with `.`, `!`, or `?` (after stripping trailing whitespace), AND (b) it contains at least one alphabetic character (filters out punctuation-only fragments).

**Check:**

For each assertion sentence: if `r'\[\^ev-\d{6}\]'` matches anywhere in the sentence, OK. Otherwise emit `VerifierFailure(kind="uncited-claim", variant, section_path, detail=f"Sentence ends '...' but has no [^ev-*] cite")` where `'...'` is the first ~80 chars of the sentence for human readability.

**Known v0 false positives (accepted trade-off):**

- Abbreviations like `e.g.`, `Mr.`, `etc.` trigger spurious sentence boundaries.
- URLs containing `.` (e.g., `example.com/path.html`) may trigger boundary splits mid-URL.

If real overnight runs surface a high false-positive density, v0.1 can swap the splitter for a smarter implementation (e.g., NLTK Punkt) — but that adds a dependency, which P4 of the parent design rules out for v0.

### 3.3 `verify_cite_resolution`

Operates on ALL sections (not just `decided`) — a stray cite in an unresolved section is still wrong.

For each `[^ev-NNNNNN]` regex match in the body:

1. Construct `ev_path = evidence_root / f"ev-{NNNNNN}.md"`.
2. If `ev_path.exists()` is False: emit `VerifierFailure(kind="dangling-cite", detail=f"ev-NNNNNN not found at {ev_path}")`.
3. Else: read the file, find frontmatter between `+++` fences, parse via `tomllib`. If parse fails OR the file has no `+++` fence: emit `VerifierFailure(kind="dangling-cite", detail=f"ev-NNNNNN has malformed frontmatter")`.
4. Else: check for a non-empty `superseded_by` field. If present with a non-empty string value: emit `VerifierFailure(kind="superseded-cite", detail=f"ev-NNNNNN is superseded_by={value!r}")`.

A single cite that fails both existence AND parsing only emits one failure (existence wins).

### 3.4 `verify_excerpt_match`

Operates on ALL sections.

For each `[^ev-NNNNNN]` regex match in the body:

1. Locate the **needle**: the sentence containing the cite. Algorithm:
   - Let `pos` = the cite's character position in the body.
   - Scan backward from `pos - 1` for the most recent `.`, `!`, `?`, or `\n\n` (paragraph break) or start-of-body. The character AFTER that boundary is the needle's start.
   - Scan forward from the cite's end position for the next `.`, `!`, `?`, or `\n\n` or end-of-body. The character at that boundary (inclusive of the punctuation) is the needle's end.
   - Strip leading/trailing whitespace.
   - If the needle is empty (rare edge case), fall back to the entire paragraph containing the cite.

2. Load the cited evidence file:
   - `ev_path = evidence_root / f"ev-{NNNNNN}.md"`.
   - If `ev_path.exists()` is False: this case is owned by `verify_cite_resolution`. To avoid double-reporting, `verify_excerpt_match` silently skips (do not emit; cite-resolution will catch the dangling cite).
   - Else: parse frontmatter via `tomllib`. If parse fails: silently skip (cite-resolution catches malformed evidence).
   - Else: read `excerpt` field. If the field is absent or empty: emit `VerifierFailure(kind="excerpt-mismatch", detail=f"ev-NNNNNN frontmatter has no excerpt field")`.

3. Normalize both needle and excerpt via `_normalize_text`.

4. Compute `ratio = difflib.SequenceMatcher(None, normalized_needle, normalized_excerpt).ratio()`.

5. If `ratio < threshold`: emit `VerifierFailure(kind="excerpt-mismatch", detail=f"ev-NNNNNN excerpt match ratio={ratio:.3f} below threshold {threshold}", excerpt_diff=<short unified diff>)` where the diff is produced by `difflib.unified_diff(normalized_excerpt.splitlines(), normalized_needle.splitlines(), lineterm="", n=1)` joined into a 5-line string.

### 3.5 `_normalize_text`

Per parent design line 83 — six steps in this exact order:

1. `s = unicodedata.normalize("NFC", s)` — Unicode canonical composition first, so smart quotes are properly recognized.
2. Smart-quote / dash map. Replacements:
   - `“` (left double quote) → `"`
   - `”` (right double quote) → `"`
   - `„` (German low-9 quote) → `"`
   - `‟` (high-reversed-9 quote) → `"`
   - `‘` (left single quote) → `'`
   - `’` (right single quote) → `'`
   - `—` (em dash) → `-`
   - `–` (en dash) → `-`
3. `s = s.lower()`.
4. `s = re.sub(r'\s+', ' ', s).strip()` — collapse any whitespace run (including newlines, tabs) to a single space; strip leading/trailing.
5. Per-word leading/trailing punctuation strip: `' '.join(w.strip(string.punctuation) for w in s.split(' '))`.

The function is pure (no I/O), deterministic, and idempotent (calling it twice yields the same result as calling it once).

### 3.6 Failure ordering and aggregation

Each verifier walks all variants and all sections, collecting failures into a list, and returns a single `VerifierResult` at the end. No early exit — all failures are reported in one pass. The list is ordered by walk order: variant directory order (sorted by name), section file order within each variant (sorted by name), occurrence order within each section.

`VerifierResult.verdict` is `"pass"` if `failures` is empty, `"fail"` otherwise.

## 5. Edge cases

- **Variant directory exists but has no `doc/` subdir:** walk skips silently. Verdict `"pass"`.
- **Section file has frontmatter but no `tags` field:** treated as `tags = []`. Citation completeness does not apply (not `decided`); cite resolution and excerpt match still apply.
- **Cite to `ev-000000`:** the regex `\d{6}` matches `000000`. No special-case. The evidence file `ev-000000.md` either exists or doesn't, per the resolution check.
- **Multiple `[^ev-XXX]` for the same `XXX` in one sentence:** each match is checked separately. The cite-resolution check will emit duplicate failures if XXX is dangling — accepted v0 behavior (orchestrator can dedup if it cares).
- **Cite inside a fenced code block:** counts as a regular cite for all three verifiers (the code-block strip only applies to citation-completeness assertion detection, not to cite resolution / excerpt match). Rationale: a cite inside a code block is unusual but if present, it should still resolve.
- **Threshold = 0.0:** all matches pass. Threshold = 1.0: only verbatim matches (after normalization) pass.
- **The cited evidence file's excerpt is shorter than the needle (or vice versa):** `SequenceMatcher.ratio()` naturally degrades; this is what we want.
- **A section body is entirely a fenced code block:** after stripping, the prose body is empty; no assertions; `verify_citation_completeness` passes for that section.

## 10. What's NOT in v0 (deferred)

- **Verifier C cross-model.** Sub-project 3 (subprocess wrapper) handles the JSON contract; the round loop (sub-project 4) drives it.
- **Verifier integration with pre-commit hook.** The hook reads from staged files; the verifiers read from on-disk variants. They serve different gates and have different I/O contracts. v0.1 may extract a shared text-normalization helper into a small shared module both can import.
- **Smarter sentence splitter** (NLTK Punkt or similar). v0 accepts trade-off; revisit only if real runs show high false-positive density.
- **Per-cite confidence weighting.** The evidence file's `confidence: high|medium|low` field is ignored by Verifier B today. v0.1 may scale the threshold by confidence (e.g., high requires 0.92, low requires only 0.85).
- **Performance optimization** for repeated cites in one round. Today each `[^ev-XXX]` re-reads the evidence file. v0.2 can add a per-invocation cache.
- **`repo/` adapter for sources outside the workspace.** v0.2+.
